## 6.1 签名共振机制 ✅

### 6.1.1 设计目标

- **复杂度**: O(N+M)
- **语义共振**: 不是标签匹配或分类，任何文本都能处理
- **发现未知关联**: 通过超维干涉模式
- **能量效率**: 99% 在低能耗层过滤

### 6.1.2 核心机制：超维计算（HDC）

所有信息表示为 D=10,000 维二进制超向量。

| 操作 | 数学实现 | 作用 |
|------|---------|------|
| 绑定 (Bind) | XOR | 编码关系 |
| 捆束 (Bundle) | 逐位加法+阈值化 | 创建复合画像 |
| 相似度 | Hamming距离 | 判断共振强度 |

**关键特性**: 捆束产生干涉模式——"分布式系统"+"共识"+"容错"捆束后会与"区块链治理"产生非零相似度。

### 6.1.3 三层共振过滤 ⚠️ 第三层待恢复

```
第一层: Bloom Filter 门控     ~100ns, 过滤 ~90%    → 🔮 规模不需要
第二层: HDC 共振检测           ~1μs, 过滤剩余 ~90%  → ✅ 已实现
第三层: 深度评估（LLM）        ~10ms, 精确评估       → ⚠️ 未实现（关联 D-02）
```

**V1 决策**: 只实现第二层。第一层规模不需要，第三层用 LLM 替代。
**V2 现实**: 第三层也没实现。匹配到就直接进结晶。

### 6.1.4 编码流程 ✅

```
步骤一: 文本 → 语义嵌入（sentence-transformers, 768维浮点）
步骤二: 浮点向量 → 二进制超向量（SimHash, 10000维）
步骤三: 超向量绑定到 Agent 或消息（bundle 操作）
```

编码器可插拔：前期 sentence-transformers → 中期可换 OpenAI embedding → 后期可训练领域模型。

### 6.1.5 Agent 画像生成与演化 ✅

投影式画像生成（Design Log #003）:
```
ProfileDataSource.get_profile() → Profile Data
    → 每项编码为超向量 → bundle → Agent 画像
    = project(profile_data, lens)
```

画像不通过"融合"更新——数据源更新 + 重新投影。

### 6.1.6 共振检测流程 ✅

```
需求文本 → 编码为 H_demand → 广播
每个 Agent: similarity(H_agent, H_demand) > θ → 激活
```

### 6.1.7 传播机制

- **V1/V2**: 简单广播（集中式匹配），1000 Agent 规模足够
- **🔮 未来**: 语义 Gossip（万级以上规模）

### 6.1.10 共振阈值策略（θ 与 k* 机制）⚠️ 概念实现

> **V1 核心洞察**: θ 不是预设常数，从期望响应数 k* 计算。k* = 本质（业务参数），θ = 实现（技术参数）。

**k* 为什么更好**:
1. k* 是业务语言，θ 是技术参数
2. k* 直接关联成本
3. k* 与 N 解耦
4. k* 可从场景语义推导

**V2 实现**: `match_perspectives()` 和 `match()` 返回 top-k，本质上是 k* 的简化实现。

**V1 场景示例**:
- 黑客松（3-5人小队）→ k* = 15-20
- 创业找联创 → k* = 5-10
- 社交场景 → k* = 30-50
- 专业服务 → k* = 3-5

**🔮 V2+ 自适应策略**: 基于回声反馈自动调整 k*（Offer率 < 50% → k*减少；成功率 > 80% → k*增加）
